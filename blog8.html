In this blog I am talking about the error handling in JavaScript such as 'try', 'catch', 'throw' and 'finally'.
'try' and 'catch' comes in pair. Similar to JAVA, the 'try' statement allows you to check the block of code to be tested while in program execution and 'catch' statement is used to define the block of code if any error occurs.
The 'throw' statement when used together with 'try' and 'catch' creates custom error message. Thus, you can throw an exception. The exception can be a JavaScript String, a Number, a Boolean or an Object. In 'finally' statement, it helps user to execute the code after try and catch no matter what the result is.

The code snippet within the body is: 

   
        try { 
          if(x == "") throw "is empty";
          if(isNaN(x)) throw "is not a number";
          x = Number(x);
          if(x > 10) throw "is too high";
          if(x < 5) throw "is too low";
        }
        catch(err) {
          message.innerHTML = "Error: " + err + ".";
        }
        finally {
          document.getElementById("demo").value = "";
        }
      }
